1.클라이언트가 url 웹페이지에 있는  input 작성 후 버튼클릭
2. jsp에있는 JS함수(ajax,  Fuction() selectReply()등)실행
3. js 가 ajax 실행 url 확인
4. 톰캣 spirng mvc에 넘김
5. DispatcherServlet이 넘어온 요청을 받고
6.컨틀롤러안에있는 같은 매핑값을 찾음
7.컨트롤러 메서드실행 -> 결과를 JSON으로 응답
8.JSON(데이터)만 응답주고 , 화면은 JS가 처리함
( jsp로 응답 X -> 페이지이동 , js로 응답 -> jsp는 그대로있고 화면일부만수정)
그럼 컨트롤러가 제이슨응답을했을때 jsp 안에있는 js만 건드리니깐
jsp 에게 응답을하는건 아니라고 보는거다

✔️ JSP는 자바 객체를 사용하고
✔️ JS는 그 자바 객체를 모름
✔️ 그래서 자바 객체 → JSON으로 바꿔서 넘겨줘야 JS가 쓸 수 있음
✔️ 그리고 script 안에 JS 쓰면 서버에서 가져온 모든 데이터가 거기 있진 않음
➡️ 그래서 필요한 데이터만 전달하려고 JSON을 쓰는 거다!

JSP	서버에서 자바 객체로 데이터 처리 → HTML로 렌더링해서 브라우저에 보여줌
JS (<script>)	브라우저에서 실행됨. 자바 객체에 직접 접근 못 함
JSON	서버와 클라이언트(JS) 간에 데이터만 주고받는 형식
왜 JSON 씀?	📌 자바 객체를 JS에서 쓸 수 있게 바꾸려고, 그리고 필요한 데이터만 깔끔하게 전달하려고
